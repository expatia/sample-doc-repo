# User Interface (UI) Components

This document lists all the significant User Interface components used in the product application suite.


| UI&nbsp;Component | Description |
| ------------ | ----------- |
| [AppDrawer](./app-drawer.md) | Displays all applications available to the user from the application suite provided by the platform. |
| [Canvas](./canvas.md) | Provides a canvas on which to position cards, widgets and views and logically associate them with one another. |
| DashboardLayout | This component provides positioning and resizig support for dashboard widgets. |
| Footer | The common footer displayed in all applications. This will provide information about intellectual property, contact information as well as a means of reporting issues and feedback about the product. |
| Header | This is the application bar shown a the top of the display. This has icon buttons for the application drawer as well as notifications, alerts, messages and warnings. |
| Layouts | Layout components are used across applications with the application suite to guarantee consistency in the placement and styling of headers, footers, navbars, side drawers and so on. |
| List | The common list component that guarantees consistency of lists across all applications within the application suite.
| Navbar | The common navbar component shown down the left hand side of each application. This is contained within a left hand drawer and has two components: a primary nav component (with icons) and a secondary nav component (a list that will priamrily rely on meaninful textual labels although may also show associated icons). |
| Overlay | An overlay designed to show more detailed content about an item in the main display over which it is shown. |
| Primary Nav | The primary nav component is usually shown in a collapsed iconified state. |
| Secondary Nav | The secondary nav gives additional navigation options and may support sorting and searching of navigation menu items. |
| Side drawer | This is a drawer displayed on the right hand side of the window. it is used to display additional tools that are relevant to the main view being displayed. |
| Table | A filterable, selectable, pageable and sortable table to display entity data |
| Toast | An informational UI element that is temporarily displayed to confirm that an event has occurred. Typically this will pop up from the bottom of the screen. |
| Tooltip | Simlar to an Overlay in that it appears over an item when a pointing device hovers over that item. The tooltip gives a hint of what the item may be used for. The ovverlay gives more detailed information about that item. |
| Widgets | Widgets may be displayed inside other components. Typically widgets are displayed inside dashboard components or reports or both. |
