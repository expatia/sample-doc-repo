# Platform Documentation Set

Welcome to the product documentation set. This is intended for internal consumption by authorized company staff members.

## Purpose

The purpose of this document set is to document our hardware, software, external communication channels and digital services and how these interact to add value to our business.

## Overview of Documentation

This documentation set is divided into a number of sections. This is in order to help our teams to find what we need more efficiently. Each section of the documentation includes links to the other sections where this may be useful.

Examples of this include:

- Links to our other products
- Links to explain the context in which our products are used and first experienced
- Links to explain what standards, constraints, regulations, laws and policies apply to a particular piece of software or hardware that we're currently reading about.
- Links between high-level architecture documents and product software architecture documents

## Documentation Set Structure

Our documentation set is structured as described in the table below.


| Section | Description |
| ------- | ----------- |
| [Analysis](./analysis/README.md) | Analysis documentation driving requirements, product quality and live service delivery policy |

| [Content](./content/README.md) | Documentation pertaining to the use of content within our software products and content about our documentation itself. This section also describes how digital and social media channels integrate with our software products and how this is used to drive sales, augment perceived product value and brand value in our target markets and build commercial partnerships in accordance with our corporate vision and business strategy.|
| [Control](./control/README.md) | Project control documentation. This includes documentation related to product planning, roles and responsibilities. |
